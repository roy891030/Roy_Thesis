\subsection{資料來源}
\label{subsec:data_source}
\FloatBarrier

資料取自 \textbf{TEJ 台灣經濟新報（Taiwan Economic Journal, TEJ）}之台灣股票市場日頻資料，並以 CSV 形式匯出以供後續處理與模型訓練使用。本文整體實驗資料之最長樣本期間為 2019-09-16 \(\sim\) 2025-09-12，共 1460 個交易日，作為後續短期、中期與長期三種樣本視窗的母體資料範圍。資料以「交易日\(\times\)股票」為觀測單位，至少包含日期（Date）、股票代碼（Stock ID）與收盤價（Close）等必要欄位；同時保留開高低收（OHLC）、成交量／成交值、市值、流通股數、估值指標（如 PB、PS）與產業分類等資訊，以支援特徵建置與圖結構構建。樣本期間由參數 \texttt{start\_date} 與 \texttt{end\_date} 控制，並以台股交易日序列作為時間索引基準。

\subsection{資料預處理}
\label{subsec:preprocess}
\FloatBarrier

資料預處理主要涵蓋資料格式統一、特徵與標籤計算、以及張量化整理，並以一致規則處理缺失值以降低訓練偏誤。整體流程如演算法 \ref{alg:preprocessing} 所示。

\begin{algorithm}[H]
\caption{資料預處理流程}
\label{alg:preprocessing}
\begin{algorithmic}[1]
\State 讀取 TEJ 匯出之日頻資料（至少包含 Date、Stock ID、Close；其餘欄位供特徵與圖結構使用）
\State 依 Stock ID 與 Date 排序；進行日期格式與數值型態轉換，並做基本缺漏/異常檢核
\State 依價格與量能序列計算技術特徵（含 TA-Lib 指標），形成 \texttt{feature\_cols}
\State 計算原始標籤: $y^{raw}_t = \frac{P_{t+k} - P_t}{P_t}$ \Comment{$k$ 日未來報酬率（forward return）}
\State 將標籤做每日截面去均值，得到訓練標籤 $y_t = y^{raw}_t - \bar{y}^{raw}_t$
\State 由產業欄位建構產業圖 $E_{ind}$；由股票清單建構全市場圖 $E_{uni}$
\State 建立特徵張量 $F^t \in \mathbb{R}^{T \times N \times F}$、去均值標籤張量 $y^t \in \mathbb{R}^{T \times N}$ 與原始標籤張量 $(y^{raw})^t \in \mathbb{R}^{T \times N}$
\State 對每個交易日之橫斷面特徵做 z-score 標準化；特徵 NaN/Inf 以 0 取代，標籤 NaN 保留並於訓練/評估以 mask 排除
\State 模型輸入端仍額外使用 Batch Normalization（BatchNorm）以穩定訓練
\end{algorithmic}
\end{algorithm}

\FloatBarrier

\textbf{缺失值處理（Missing values）：}不對缺失值進行插補，以避免引入主觀假設。特徵層面將 NaN/Inf 以 0 取代以維持張量完整性；標籤層面保留 NaN，並在訓練與評估時以有效樣本遮罩（mask）排除，使模型僅在可觀測標籤上學習。

\textbf{特徵標準化（Normalization）：}實作上，資料前處理階段先對每個交易日的股票橫斷面特徵做 z-score 標準化，以統一不同特徵在同日截面上的尺度：
\begin{equation}
x^{xsec}_{i,t,f} = \frac{x_{i,t,f} - \mu_{t,f}}{\sigma_{t,f} + \epsilon},
\label{eq:xsec_zscore}
\end{equation}
其中 \(\mu_{t,f}\) 與 \(\sigma_{t,f}\) 分別為交易日 \(t\) 上第 \(f\) 個特徵的截面平均數與截面標準差，\(\epsilon\) 為數值穩定項。完成張量化後，模型輸入層仍再使用 \textbf{Batch Normalization（BatchNorm）} 進行第二層正規化，以降低不同 batch 間的分布波動：
\begin{equation}
x_{normalized} = \frac{x - \mu_{batch}}{\sqrt{\sigma^2_{batch} + \epsilon}}
\label{eq:batchnorm}
\end{equation}
其中 $\mu_{batch}$ 與 $\sigma^2_{batch}$ 為 mini-batch 統計量，$\epsilon$ 為數值穩定項（預設 $10^{-5}$）。

\textbf{標籤定義（Label definition）：}原始標籤先定義為未來 $k$ 日報酬率（forward-$k$ return）：
\begin{equation}
y^{raw}_t = \frac{P_{t+k} - P_t}{P_t}
\label{eq:return_raw}
\end{equation}
其中 $P_t$ 為時間 $t$ 收盤價，$k$ 為預測視窗（預設 $k=5$）。在訓練與預測評估時，為降低單日市場整體漲跌對截面排序學習的干擾，本文進一步將原始標籤做每日截面去均值，得到
\begin{equation}
y_{i,t} = y^{raw}_{i,t} - \frac{1}{N_t}\sum_{j=1}^{N_t} y^{raw}_{j,t}.
\label{eq:return_demean}
\end{equation}
亦即，模型學習與 RMSE / MAE / IC / ICIR 等預測指標所對應者，是去除市場當日平均報酬後的相對報酬排序；而原始 forward-$k$ 日報酬 \(y^{raw}\) 會另行保存，供後續投資組合回測與 0050 基準比較使用。



\subsection{資料集劃分}
\label{subsec:split}
\FloatBarrier

為避免模型訓練過程產生未來資訊滲入（look-ahead bias），本文採用時間序列切分（time-series split）進行訓練、驗證與測試集的三段劃分。具體而言，資料依日期由早至晚排序後，先將最後 20\% 交易日保留為測試集（Test）；再自前段 80\% 區間尾端切出 \texttt{val\_ratio}=0.1 的驗證集（Validation），其餘作為訓練集（Train）。因此，就全樣本而言，整體比例約為 72\% 訓練、8\% 驗證、20\% 測試，確保模型僅能利用過去資訊進行訓練、調參與最終評估。

此外，為檢驗模型在不同樣本長度下之穩健性，本文設計短期、中期與長期三種樣本視窗，並皆採用相同的三段時間切分方式。圖 \ref{fig:train_test_split} 依實際日期位置展示三種視窗之 Train / Validation / Test 切分；圖 \ref{fig:train_val_test_split_bar} 則以橫向長條補充各視窗的日數配置，以利直觀比較不同樣本長度下可用資訊量與測試期間差異。

\begin{itemize}
    \item \textbf{短期資料：}2019-09-16 $\sim$ 2020-12-31（共 319 個交易日；訓練 230 日、驗證 25 日、測試 64 日）
    \item \textbf{中期資料：}2019-09-16 $\sim$ 2022-12-31（共 809 個交易日；訓練 583 日、驗證 64 日、測試 162 日）
    \item \textbf{長期資料：}2019-09-16 $\sim$ 2025-09-12（共 1460 個交易日；訓練 1052 日、驗證 116 日、測試 292 日）
\end{itemize}

\textbf{產業分類（Industry classification）：}產業欄位採用 TEJ 提供之「TEJ產業\_名稱」。該分類於樣本期間內具相對穩定性，適合用於建構產業層級圖結構；實驗中不額外進行主觀重分群。

\subsection{圖結構設計與建構流程}
\label{subsec:graph_build}
\FloatBarrier

\textbf{Static 產業圖（Static Industry Graph）：}
static 口徑以「同產業股票完全連結」為原則，將同一產業內之股票視為相互連結，
並為每檔股票加入自環（self-loop），以捕捉產業內共通訊號與個股自身資訊。
圖 \ref{fig:industry_graph_demo} 以三個示例產業為例，展示同產業節點形成高密度子圖（cluster），且每個節點皆具有自環結構。

    \label{fig:industry_graph_demo}
\end{figure}
\FloatBarrier

\textbf{Static 全市場圖（Static Universe Graph）：}
static 全市場圖採完全圖（complete graph）設計，所有股票之間皆存在連結，並同樣包含自環。
此設計目的在於表徵跨產業之市場共同因子影響，使模型可在截面層級整合全市場訊息。
圖 \ref{fig:universe_graph_demo} 顯示完全連結圖結構下，節點之間呈現高度稠密的連線關係。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.85\linewidth]{img/market_graph.png}
    \caption{全市場圖示意（Universe Graph）：完全連結並包含自環（self-loop）。在此圖結構下，每個節點可與所有其他節點交換訊息，以表徵跨產業的市場共通因子影響。}
    \label{fig:universe_graph_demo}
\end{figure}
\FloatBarrier
\textbf{Dynamic weighted graph：}
除 static binary graph 外，本文亦建立 dynamic weighted graph。dynamic graph 對每一交易日 \(t\) 使用截至當日的一日報酬 rolling correlation matrix，預設 lookback 為 60 個交易日、最少有效觀測數為 20。產業圖僅在同產業候選集合內保留每個 target stock 絕對相關最高的 \texttt{industry\_top\_k}=20 個非自身鄰居；全市場圖則在全市場候選集合內保留 \texttt{universe\_top\_k}=40 個非自身鄰居。每個節點仍保留自環。dynamic graph 的 edge attributes 為二維向量 \([\lvert \rho_{ij,t}\rvert, \rho_{ij,t}]\)，分別代表絕對相關強度與帶符號相關方向。

\textbf{表示方式（Representation）：}
圖結構以 PyTorch Geometric（PyG）之 \texttt{edge\_index} 表示，其為一個形狀為 $[2, E]$ 的整數張量，
其中 $E$ 為邊（edges）數量。具體而言，\texttt{edge\_index} 的每一欄對應一條邊，
第一列為起點節點（source / start node），第二列為終點節點（target / end node）。
在 dynamic graph 口徑下，模型另讀取與 \texttt{edge\_index} 對齊的 \texttt{edge\_attr}，使 GATConv 可同時使用節點特徵與邊權重資訊。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.85\linewidth]{img/edge_index_demo.png}
    \caption{\texttt{edge\_index} 表示方式示意圖：左側為小型圖結構（節點 A、B、C），右側為其對應之 \texttt{edge\_index}。
    \texttt{edge\_index} 以 $[2, E]$ 形式紀錄所有邊，每一欄代表一條由起點（start）指向終點（end）的連結關係。}
    \label{fig:edge_index_demo}
\end{figure}
\FloatBarrier

\textbf{建構流程（Construction）：}由輸入 CSV 讀取產業欄位並依股票代碼分組建立 static 產業圖；若產業欄缺漏，則以單一產業處理。static 全市場圖依股票清單建立完全連結。dynamic graph 則依每日 rolling correlation 生成產業與全市場兩組每日 \texttt{edge\_index} 與 \texttt{edge\_attr}。上述圖結構與特徵張量共同保存為 artifacts，以支援訓練與評估之重複使用，並確保 static 與 dynamic graph 使用相同時間窗與資料切分。

\section{模型設計}
\label{sec:models}
% =========================================================

\subsection{非圖基準模型}
\label{subsec:baseline_models}

本文保留三個不使用圖結構的基準模型。Linear baseline 採 Ridge regression，作為線性可解釋基準；XGBoost baseline 使用 gradient boosting tree 捕捉 tabular features 中的非線性與交互作用；LSTM baseline 以 lookback 序列輸入，作為時間序列模型基準。三者皆使用相同的特徵張量與時間序列切分，並僅在 static graph 口徑下列入統一比較，因其模型本身不消耗 \texttt{edge\_index} 或 \texttt{edge\_attr}。

\subsection{Factor ablation models}
\label{subsec:factor_ablation_models}

圖／因子模型採用共同的 \texttt{FactorGraphAblation} 架構，以便在相同 encoder、decoder 與訓練設定下比較不同圖訊息與中性化設計的增量效果。令每個交易日之輸入特徵為 \(F^t \in \mathbb{R}^{N_t \times F}\)。模型先經 BatchNorm 與 MLP encoder 取得個股語境表示：
\begin{equation}
C^t = \mathrm{MLPEncoder}(\mathrm{BatchNorm}(F^t)).
\end{equation}
當模型使用圖結構時，產業圖與全市場圖分別透過 GATConv 學習影響表示：
\begin{equation}
H_I^t = \mathrm{GAT}_{I}(C^t, E_I^t), \qquad
H_U^t = \mathrm{GAT}_{U}(\cdot, E_U^t),
\end{equation}
其中 \(E_I^t\) 與 \(E_U^t\) 在 static graph 下為固定 binary edges，在 dynamic graph 下則為每日 rolling correlation 加權邊並帶有 edge attributes。

本文比較六種 factor variants：
\begin{itemize}
    \item \textbf{MLP：}僅使用 \(C^t\)，不使用任何圖結構。
    \item \textbf{GAT-Industry：}使用 \([C^t \, || \, H_I^t]\)，檢驗產業圖聚合是否提供增量資訊。
    \item \textbf{GAT-Universe：}使用 \([C^t \, || \, H_U^t]\)，檢驗全市場圖聚合是否提供增量資訊。
    \item \textbf{GAT-TwoGraph：}使用 \([C^t \, || \, H_I^t \, || \, H_U^t]\)，但不做 subtraction neutralization，用以分離「雙圖聚合」與「中性化」效果。
    \item \textbf{DMFM-IndNeutral：}先以產業圖學得 \(H_I^t\)，再形成 \(C_I^t = C^t - H_I^t\)，並使用 \([C^t \, || \, C_I^t]\)。
    \item \textbf{DMFM-Full：}完整執行 \(C_I^t = C^t - H_I^t\)，再以 \(C_I^t\) 輸入全市場圖得到 \(H_U^t\)，形成 \(C_U^t = C_I^t - H_U^t\)，最後使用 \([C^t \, || \, C_I^t \, || \, C_U^t]\)。
\end{itemize}

上述各 variant 之輸入表示再經 decoder MLP 輸出每檔股票的深度因子或預測分數。此設計使本文能清楚辨識績效改善來自單純非線性特徵編碼、產業圖、全市場圖、雙圖聚合，或分層中性化。

\subsection{因子注意力與解釋性輸出}
\label{subsec:factor_attention}

除不使用圖的傳統 baseline 外，factor variants 均可啟用 factor attention module。實作上，模型對 BatchNorm 後的原始特徵學習特徵層注意力權重 \(a^t\)，並以 softmax 使同一股票的特徵權重可比較。注意力加權後的特徵再透過可學習 projection head 估計 \(\hat{f}^t\)，作為損失函數中的解釋性約束來源。注意力權重不應被解讀為嚴格因果效果，而是用於觀察模型在不同 window、graph mode 與 variant 下偏好的訊號類型。

\textbf{超參數配置（Hyperparameters）：}本文主 pipeline 之 factor variants 設定為 \texttt{hidden\_dim}=64、\texttt{heads}=2、\texttt{dropout}=0.1、\texttt{learning rate}=1e$-$4、\texttt{weight decay}=0.01、$\lambda_{attn}=0.05$、$\lambda_{IC}=1.0$；訓練週期依樣本視窗調整為 Short = 50、Medium = 100、Long = 100，對應之 \texttt{patience} 分別為 20、30、30。full unified pipeline 使用 \texttt{graph\_modes=static,dynamic}，baseline 預設僅在 static 口徑下訓練與評估。

\section{損失函數}
\label{sec:loss}
% =========================================================

\subsection{總損失函數}
factor variants 採用相同的排序導向損失函數，使 MLP、GAT variants 與 DMFM variants 能在一致口徑下比較：
\begin{equation}
L = \lambda_{attn} \cdot d(f,\hat{f}) + \lambda_{IC} \cdot (1 - IC) - \lambda_b \cdot b
\label{eq:loss}
\end{equation}
其中 $d(f,\hat{f})$ 為注意力估計誤差，實作上以標準化後深度因子與注意力估計因子之均方差衡量；$IC$ 為資訊係數（Information Coefficient, IC），$b$ 為截面回歸得到的因子收益項。實作中 $\lambda_b$ 固定為 0.01，以降低因子收益項之不穩定性。若關閉 factor attention，則注意力估計誤差項為 0。

\textbf{IC（Information Coefficient）：}本文主文以 Pearson correlation 計算，基於每個交易日之截面預測與真實 forward-$k$ 日報酬衡量訊號有效性。採用 Pearson 的理由是需與式 \ref{eq:loss} 中以 IC 為目標的連續可微優化口徑一致，並保留對預測強弱幅度的敏感性。

\textbf{優化器（Optimizer）：}factor variants 使用 AdamW，並加入 weight decay 以抑制過度擬合；Linear、XGBoost 與 LSTM 則分別依其模型特性使用 Ridge regression、XGBoost squarederror objective 與 MSE loss。

\section{評估指標}
\label{sec:metrics}
% =========================================================

\subsection{預測品質指標}
\textbf{誤差指標（Error metrics）：}本文先以 MSE、RMSE 與 MAE 衡量模型對未來報酬數值的貼近程度。其中，MSE 反映誤差平方後的平均值，對較大偏差更敏感；RMSE 為 MSE 開根號後的結果，與原始報酬尺度一致；MAE 則以絕對誤差平均衡量整體偏離程度，較不受極端值放大影響。

\textbf{IC / Daily IC / ICIR：}令 \(\hat{y}_{i,t}\) 與 \(y_{i,t}\) 分別表示股票 \(i\) 在交易日 \(t\) 的預測值與實際 forward-$k$ 日報酬，則單日截面資訊係數定義為
\begin{equation}
IC_t = \mathrm{Corr}\!\left(\hat{y}_{1:N_t,t}, y_{1:N_t,t}\right).
\end{equation}
若將所有交易日的 \(IC_t\) 進一步彙整，則
\begin{equation}
Daily\ IC = \frac{1}{T}\sum_{t=1}^{T} IC_t,
\qquad
ICIR = \frac{\frac{1}{T}\sum_{t=1}^{T} IC_t}{\mathrm{Std}(IC_t)}.
\end{equation}
其中 \(T\) 為測試期間交易日數，\(\mathrm{Std}(IC_t)\) 為逐日 IC 的樣本標準差。本文並另行回報整體樣本之 Pearson correlation，作為跨全測試集彙總後的 IC。

\textbf{方向判斷正確率（Directional Accuracy, DirAcc）：}DirAcc 用以衡量模型對報酬方向的判斷能力，即預測值與實際報酬在符號上是否一致。相較於 IC 與 ICIR 側重排序品質，DirAcc 提供另一個較直觀的方向性判準。

\textbf{產業中性 IC（Industry-neutral IC）：}除整體 IC 外，本文對具產業圖與產業標籤之 factor variants 另行計算產業中性 IC，以檢查排序能力是否僅來自產業共同波動。做法為先在每交易日、每產業內分別對預測與真實報酬去均值（industry de-meaning），再對殘差計算 Pearson correlation。令 $\hat{y}_{i,t}$ 與 $y_{i,t}$ 分別為股票 $i$ 在日 $t$ 的預測與真實報酬，$g(i)$ 表示產業別，則
\begin{align}
\tilde{\hat{y}}_{i,t} &= \hat{y}_{i,t} - \frac{1}{|G_{g(i),t}|}\sum_{j \in G_{g(i),t}} \hat{y}_{j,t}, \\
\tilde{y}_{i,t} &= y_{i,t} - \frac{1}{|G_{g(i),t}|}\sum_{j \in G_{g(i),t}} y_{j,t},
\end{align}
其中 $G_{g(i),t}$ 為日 $t$ 時產業 $g(i)$ 的有效股票集合（排除標籤為 NaN 者）。產業中性 IC 定義為
\begin{equation}
IC^{\text{ind-neutral}}_t = \text{Corr}\left(\tilde{\hat{y}}_{\cdot,t}, \tilde{y}_{\cdot,t}\right).
\end{equation}
本文並同樣彙整其逐日平均與穩定度，作為補充檢視圖模型是否在排除產業共同成分後，仍能保留具經濟意義的個股相對強弱訊號。

\subsection{投資組合績效指標}
\textbf{年化報酬率（Annualized return）：}以每次再平衡選取預測分數最高之 \texttt{top\_pct}=0.10 股票等權做多，計算截面平均報酬作為策略報酬，並依再平衡頻率年化。本文預設每 5 個交易日再平衡一次（\texttt{rebalance\_days}=5），故策略統計所依據者為 5 日持有期報酬序列。

\textbf{Sharpe Ratio：}以平均報酬除以波動後再乘年化因子，衡量風險調整後報酬。

\textbf{勝率（Hit ratio）：}正報酬期間比例，用以衡量策略穩定性。若策略在第 \(t\) 次再平衡後之單期報酬為 \(r^{p}_t\)，則勝率定義為
\begin{equation}
Hit\ Ratio = \frac{1}{T}\sum_{t=1}^{T} \mathbf{1}(r^{p}_t > 0),
\end{equation}
其中 \(\mathbf{1}(r^{p}_t > 0)\) 為指示函數：當第 \(t\) 期投資組合報酬 \(r^{p}_t\) 大於 0 時，其值為 1；否則為 0。

\subsection{基準比較與流程摘要}
\textbf{台灣 50 ETF 基準：}以 0050 之 forward-$k$ 日報酬作為市場基準，與策略累積報酬比較。

評估流程為：載入 artifacts 與模型權重後，先以去均值標籤計算預測層指標，再以另存之原始 forward return 進行投資組合回測與 0050 基準比較；並輸出 Daily IC、IC 分佈、累積報酬、風險報酬散點、dynamic-static delta 與注意力特徵頻率等視覺化結果。統一實驗輸出保存於 \texttt{runs\_unified}，主文表格與附錄均以 \texttt{runs\_unified/analysis\_report} 之彙整結果為準。另需說明，目前正式評估流程已不再將 naive-zero 預測列為主文比較指標；若歷史結果檔中仍保留相關欄位，應理解為舊版評估流程的遺留資訊。
